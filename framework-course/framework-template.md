# Framework Build Template v1.0
### Use this template to structure your first framework

---

## Instructions

Fill in each section for your own framework candidate. Don't aim for perfection on the first pass, the goal is to get a complete v1.0 skeleton, even a rough one. You'll refine this across later modules.

Where you see *[bracketed prompts]*, replace them with your own content. Where you see guidance text in italics, that's there to help you think, delete it once you've filled in your section.

## 3. Output Structure

*Define exactly what this framework produces. Not "an analysis", but the specific shape of the deliverable. Choose one primary format and describe it.*

**Output Format:** *[e.g. "A 0-100 score across 4 weighted categories, plus a one-paragraph verdict" or "A 2x2 matrix plotting urgency against confidence" or "A three-tier classification: Proceed / Proceed with Conditions / Do Not Proceed"]*

**Output Components:**

## 4. Governance Layer

*This is what separates a framework from a checklist. Define how your framework handles weak, missing, or contradictory evidence.*

**Minimum Evidence Threshold:** *[What's the minimum information needed before this framework can produce a reliable output? What happens if that threshold isn't met?]*

**Confidence Indicator:** *[How does your output communicate how confident the assessment is? e.g. "Each output includes a confidence rating: High / Medium / Low, based on how many of the core questions had clear answers."]*

**Conflict Handling:** *[What happens when two pieces of evidence point in different directions? Does the framework flag this, weight certain evidence more heavily, or require additional input before proceeding?]*

**Refusal/Escalation Condition:** *[Is there a scenario where this framework should explicitly NOT produce a verdict, and instead flag that the situation needs human judgment or more data? Define it.]*

---

## 5. Translation Layer

*Every framework output needs to be usable by someone who didn't build it. Define how technical output becomes a decision-ready message.*

**Primary Audience for Output:** *[Who reads the final output? A hiring manager? A client executive? Yourself, six months from now?]*

**Translation Rule:** *[What's the one-sentence version of the output that this audience needs to hear? e.g. "Regardless of the detailed score, the output always opens with: 'Recommendation: [X], because [primary driver].'"]*
